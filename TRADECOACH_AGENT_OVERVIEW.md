# TradeCoach — Agent Overview

> AI 기반 트레이딩 저널 + 복기 코치

---

## 한 줄 소개

내 Bybit 거래내역을 자동 분석해서 반복 실수를 진단하고,
ICT 이론 기반 맞춤 코칭으로 트레이더의 행동 패턴을 교정하는 AI 코치.

---

## 기존 분석 도구와의 차이

| 항목 | 기존 성과 분석기 | TradeCoach |
|------|--------------|-----------|
| 관심 대상 | 수익률 숫자 | 반복 실수 패턴 |
| 핵심 질문 | "얼마 벌었나?" | "왜 같은 실수를 반복하는가?" |
| 결과물 | 통계 리포트 | 내일 실행할 규칙 1개 |
| 메모리 | 없음 | 세션 간 약점 누적 추적 |
| 코칭 | 없음 | ICT 개념 기반 맞춤 코칭 |

## 현재 핵심 기능 (v2.0)

### 1. 거래내역 자동 수집
Bybit API로 실제 거래내역을 자동으로 가져옵니다.
API 키가 없으면 샘플 데이터로 기능을 체험할 수 있습니다.

### 2. KPI 기반 실력 진단
전통적인 손익비(R:R) 대신 실제 데이터로 계산 가능한 4가지 KPI:

| KPI | 의미 | 경고 기준 |
|-----|------|---------|
| 승률 | 수익 거래 비율 | 40% 미만 |
| 평균 수익률 | 투자금 대비 평균 수익 % | 1% 미만 |
| 기대값 | 전략의 장기 수익성 | 0 이하 |
| 손절 일관성 | 손절 규율 (낮을수록 좋음) | 2.0 초과 |

### 3. ICT 개념 기반 약점 코칭
25개 ICT 개념 사전 기반으로 약점을 진단하고 개선 방법을 제시합니다.
미등록 약점은 ICT / 심리 / 데이터 패턴 3가지로 자동 분류합니다.

### 4. 세션 간 메모리
매 세션마다 약점을 누적 저장합니다.
"이번 세션에서도 손절 불규칙이 감지됐습니다. 3회 연속입니다."
같은 장기 추적이 가능합니다.

### 5. 내일의 규칙
분석 결과를 바탕으로 내일 당장 실행할 수 있는 규칙 1개를 제시합니다.
> "BTC 셋업은 런던 세션에서만 진입할 것"

## 예상 사용 시나리오

```
[오늘 거래 후 복기]

앱 실행
  → Bybit에서 오늘 거래내역 10건 자동 수집
  → KPI 분석: 승률 40% / 수익률 0.8% / 기대값 -0.3% / 손절 일관성 2.4
  → 약점 감지: 기대값_음수 · 손절_불규칙 (3회 연속)
  → ICT 코칭:
      손절 불규칙은 감정 매매의 신호입니다.
      ICT 관점에서 손절은 OB 또는 FVG 반대편 구조적 레벨에 고정해야 합니다.
  → 내일의 규칙:
      진입 전 손절가를 먼저 설정하고
      고정 리스크($5) 기준으로 포지션 크기를 계산한 후 진입할 것
  → 세션 저장 완료
```

## 기술 구조

```
Bybit API
  ↓
LangGraph 파이프라인
  ├── 데이터 수집 (new_data_check → bybit_fetch)
  ├── 전처리 (preprocess)
  ├── 매매일지 작성 (journal_write)
  ├── KPI 분석 (journal_analysis → weakness_detect)
  ├── 성과 요약 (performance_analysis)
  └── 코칭 생성 (backtest_coach)
  ↓
SQLite 메모리 (세션 간 약점 누적)
  ↓
Streamlit UI
```

## 차별점

**1. 손익비 없이도 작동하는 KPI**
Bybit API에 손절가 데이터가 없다는 현실적 제약을 수익률·기대값·손절 일관성으로 해결했습니다.

**2. 반복 약점 추적**
단발성 분석이 아니라 세션을 거듭할수록 반복 패턴을 추적합니다.

**3. ICT 이론 기반**
일반적인 통계 분석이 아니라 ICT(Inner Circle Trader) 이론에 기반한 약점 분류와 코칭을 제공합니다.

**4. 실행 가능한 결과**
"손익비가 낮습니다"가 아니라 "내일 이렇게 하세요"로 끝납니다.

## Future Vision

v2.0 이후 TradeCoach는 아래 방향으로 진화합니다:

**v2.1 — Replay Foundation**
거래 시점의 캔들 데이터를 복원해서 "그때 차트가 어떤 상태였는지" 맥락을 제공합니다.

**v2.2 — ICT Detector**
FVG, Order Block, MSS, Liquidity Sweep을 rule-based로 자동 탐지합니다.

**v2.3 — Replay Coach**
"왜 MSS 확인 전에 진입했나요?" 같은 candle-by-candle 복기 코칭을 제공합니다.

**v3 — A+ Setup Memory**
개인별 우수 셋업을 저장하고 "이번 진입은 과거 A+ 셋업 대비 OB retest가 부족했습니다"처럼 비교합니다.
